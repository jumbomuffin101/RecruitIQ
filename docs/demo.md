# RecruitIQ Demo Script

Target duration: about 3 minutes.

1. **Sign in and select or create a workspace.** Start on the dashboard to establish that RecruitIQ is a multi-workspace hiring application, not a static candidate list.
2. **Create a Software Engineer job.** Point out the structured requirements and the 100-point category rubric. Explain that requirements are versioned and a rubric change makes older evaluations visibly stale rather than silently rewriting them.
3. **Add a candidate resume.** Show the intake flow, editable extracted profile, and application created for the selected role.
4. **Generate the hybrid evaluation.** Show the deterministic fit score, requirement-level outcomes, and grounded resume evidence. Explain that AI assesses semantic evidence while the rubric calculates the final score.
5. **Open Compare.** With multiple applicants, show ranked candidates, category scores, gaps, recommended actions, and the requirement matrix.
6. **Open Pipeline.** Move an application through the stages and show that the stage belongs to the candidate-job application.
7. **Generate an interview scorecard.** Expand a completed criterion to show the requirement-linked question, interviewer evidence, progress, and resume-to-interview validation summary.
8. **Close on Architecture.** Trace Clerk organization context through Server Actions, Prisma, and PostgreSQL. Mention the semantic-evidence path, deterministic final score, and fallback behavior.

Keep the story focused on accountable hiring: structured requirements, explainable scoring, evidence, human feedback, and organization-scoped access.
